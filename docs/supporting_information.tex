\section{Error Definitions and QC Criteria}

For a two-state benchmark, the reported gradient error is the Euclidean norm
over both electronic states and all nuclear Cartesian components,
\begin{equation}
\Delta_g =
10^3
\left[
\sum_{I=0}^{1}\sum_A\sum_{\alpha=x,y,z}
\left(
g^{\mathrm{DMRG}}_{I,A\alpha}
-g^{\mathrm{FCI}}_{I,A\alpha}
\right)^2
\right]^{1/2},
\end{equation}
with units of m$\Eh$/Bohr.  The derivative-coupling error is the corresponding
phase-aware norm,
\begin{equation}
\Delta_d =
\min_{s=\pm 1}
\left[
\sum_A\sum_{\alpha=x,y,z}
\left(
d^{\mathrm{DMRG}}_{01,A\alpha}
-s\,d^{\mathrm{FCI}}_{01,A\alpha}
\right)^2
\right]^{1/2},
\end{equation}
reported in atomic units.  Before the errors are evaluated, roots are assigned
and phase-aligned using the protocol described below.  For exact or
near-degenerate same-energy target clusters, the comparison is made after
projecting the DMRG candidate-root subspace onto the fixed FCI gauge.

Convergence plots and largest-$M$ accuracy claims use points satisfying both
conditions: (1) the minimum assigned absolute CI overlap with the FCI reference
is at least 0.98; and (2) the corresponding gradient and derivative-coupling
Lagrange response solves are converged.  All attempted points, including
low-$M$ points excluded by these numerical QC criteria, are retained in the
machine-readable JSON output.

\section{Fixed-Orbital FCI Reference Protocol}

For each molecule/basis benchmark, the reference was generated as follows.
First, RHF was converged.  Second, equal-weight SA(2)-CASSCF was converged with
the PySCF FCI active-space solver.  Third, the final SA-CASSCF orbitals were
frozen.  Fourth, the fixed-orbital singlet active-space Hamiltonian was
rediagonalized with PySCF \texttt{direct\_spin0} FCI for multiple roots.  The
two lowest spin-adapted singlet roots were selected and used to compute
reference energies, gradients, and derivative couplings.  The FCI root scan,
$\langle S^2\rangle$ diagnostics, same-energy clusters, and residual norms were
recorded as QC metadata.

DMRG calculations were then run at the selected bond dimensions in the same
frozen orbital basis.  Candidate DMRG roots were converted to PySCF-compatible
CI arrays for benchmark scoring.  Isolated target roots were assigned by maximum
absolute CI overlap with the FCI references and phase-aligned.  If the FCI scan
identified a same-energy singlet cluster within $10^{-4}$~$\Eh$, the DMRG
candidate subspace spanning that cluster was projected onto the fixed FCI gauge
before derivative errors were evaluated.

\section{FCI-Free Endpoint Check}

The production-style endpoint check evaluates DMRG roots, gradients, and
derivative couplings without loading FCI CI vectors at runtime.  The endpoint
uses energy ordering at the first point and previous-step DMRG overlap when a
previous DMRG reference is available.  For the public benchmark endpoint,
FCI data were loaded only after the DMRG calculation to score the error against
the fixed-orbital FCI references.

\section{MPS-Only Response Endpoint}

The MPS-Krylov response endpoint starts from block2 MPS roots and stores the
active-space response RHS, Krylov vectors, and Lagrange transition-density
contractions as MPS quantities.  Table~\ref{tab:si-mps-only-response}
summarizes public fixed-orbital endpoint checks.  These runs do not load FCI
CI vectors during the runtime DMRG root selection and do not store the
active-space response vector as a dense determinant array.  For anthracene,
the FCI reference is loaded only after the MPS calculation to score the
reported errors.

MPS-Krylov response settings.  Increasing the response iteration budget and

m$\Eh$.

\section{DMRG-SHARC Trajectory Smoke Tests}

The public SHARC smoke tests use the \texttt{dmrg-casscf} template method and
request NACDR coupling.  Each run completed three SHARC frames, corresponding to
steps 0, 1, and 2, and the final \texttt{QM.out} file contained the requested
Hamiltonian, dipole, gradient, derivative-coupling, phase, and runtime blocks.
These tests check the repeated-call data contract between SHARC and the
DMRG-backed PySCF object; they are not used as mechanistic photochemistry
simulations.
